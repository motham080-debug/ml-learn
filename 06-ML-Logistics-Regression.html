ประวัติศาสตร์

ปี 1838: Pierre François Verhulst นักคณิตศาสตร์ชาวเบลเยียม พัฒนา "Logistic Function" เพื่อศึกษาการเติบโตของประชากร

ปี 1944: Joseph Berkson นำ Logistic Function มาใช้ในทางสถิติและตั้งชื่อว่า "Logit"

ปี 1958: David Cox พัฒนาเป็น "Logistic Regression" ที่ใช้กันในปัจจุบัน

ความหมายของชื่อ

"Logistic" มาจากคำว่า "Logistique" ในภาษาฝรั่งเศส หมายถึง "การคำนวณ" หรือ "ตรรกะ"

"Regression" หมายถึง "การถดถอย" หรือการหาความสัมพันธ์ระหว่างตัวแปร

รวมกัน: "การถดถอยแบบลอจิสติก" - การหาความสัมพันธ์โดยใช้ฟังก์ชันลอจิสติก

🧮 ฟังก์ชันลอจิสติก (Sigmoid Function)

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

ฟังก์ชันนี้จะให้ค่าระหว่าง 0 ถึง 1 เสมอ เหมาะสำหรับการทำนายความน่าจะเป็น

💡 แนวคิดพื้นฐาน

ความแตกต่างจาก Linear Regression

Linear Regression

ทำนายค่าต่อเนื่อง (เช่น ราคา, น้ำหนัก)

ผลลัพธ์: -∞ ถึง +∞

Logistic Regression

ทำนายความน่าจะเป็น (เช่น ผ่าน/ไม่ผ่าน)

ผลลัพธ์: 0 ถึง 1

⚙️ ขั้นตอนการทำงานอย่างละเอียด

ขั้นตอนที่ 1: เตรียมข้อมูล (Data Preparation)

การทำความสะอาดข้อมูล:

  • ลบข้อมูลที่ขาดหายไป (Missing Values)
  • จัดการข้อมูลผิดปกติ (Outliers)
  • แปลงข้อมูลเป็นตัวเลข (Encoding)

การปรับมาตรฐาน:

  • Standardization (z-score)
  • Normalization (0-1 scaling)
  • Feature Selection

ขั้นตอนที่ 2: สร้างสมการเชิงเส้น (Linear Combination)

$$z = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \dots + \beta_n x_n$$

β₀ = Intercept (ค่าคงที่)

β₁, β₂, ..., βₙ = Coefficients (สัมประสิทธิ์)

x₁, x₂, ..., xₙ = Features (ตัวแปรอิสระ)

ขั้นตอนที่ 3: ใช้ Sigmoid Function

$$P(y=1) = \frac{1}{1 + e^{-z}}$$

แปลงค่า z เป็นความน่าจะเป็น

คุณสมบัติของ Sigmoid:

  • ค่าอยู่ระหว่าง 0 ถึง 1
  • เป็นเส้นโค้ง S-shape
  • Smooth และ Differentiable
  • เหมาะสำหรับ Binary Classification

ขั้นตอนที่ 4: การเรียนรู้ (Training)

Maximum Likelihood Estimation:

หาค่า β ที่ทำให้ likelihood สูงสุด

ใช้ Gradient Descent หรือ Newton-Raphson

Cost Function:

J(β) = -Σ[y·log(p) + (1-y)·log(1-p)]

Log-likelihood function

ขั้นตอนที่ 5: การทำนาย (Prediction)

Input

ข้อมูลใหม่

Process

คำนวณความน่าจะเป็น

Output

Class (0 หรือ 1)

Decision Rule: ถ้า P(y=1) ≥ 0.5 → Class 1, ถ้า P(y=1) < 0.5 → Class 0

True Positive (TP): ทำนายผ่าน และจริงๆ ก็ผ่าน

True Negative (TN): ทำนายไม่ผ่าน และจริงๆ ก็ไม่ผ่าน

False Positive (FP): ทำนายผ่าน แต่จริงๆ ไม่ผ่าน

False Negative (FN): ทำนายไม่ผ่าน แต่จริงๆ ผ่าน

🧮 สูตรการคำนวณ

เมตริกพื้นฐาน:

Accuracy = (TP + TN) / (TP + TN + FP + FN)

Precision = TP / (TP + FP)

Recall = TP / (TP + FN)

F1-Score = 2 × (Precision × Recall) / (Precision + Recall)

เมตริกขั้นสูง:

Specificity = TN / (TN + FP)

Sensitivity = TP / (TP + FN)

Log-Loss = -Σ[y×log(p) + (1-y)×log(1-p)]

AUC = ∫ TPR d(FPR)
